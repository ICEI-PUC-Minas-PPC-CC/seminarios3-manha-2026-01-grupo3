## 4. Estratégia de Desenvolvimento

### Ambiente de desenvolvimento
Vs Code, e versionamento de arquivo usando o git, e o projeto estará no GitHub para que todos possam criar uma branch e altera-lo.

### Estratégia de testes

Enviaremos o app para amigos testarem, ou então iremos na puc nos laborátorios.

### Estratégia de deploy

A aplicação será hospedada no GitHub como um projeto, assim ficará disponível a todos os participantes do grupo.

## 5. Riscos e Mitigações

| Risco | Probabilidade | Impacto | Mitigação |
| ------- | :------------: | :-------: | ---------- |
| Atraso no cronograma | Talvez | Vai atrazar| Definir margens de segurança no cronograma, priorizar entregas críticas e realizar checkpoints semanais para acompanhamento do progresso.|
| Dificuldade técnica inesperada | Baixa | Baixo| Realizar pesquisas prévias, manter documentação de apoio, buscar ajuda com colegas/comunidade e ter alternativas técnicas planejadas.|
| Comunidade indisponível para reuniões | Talvez | Alto impacto| Agendar reuniões com antecedência, oferecer horários alternativos e utilizar comunicação assíncrona (e-mail, formulários, mensagens).|
| Problemas de infraestrutura no dia da entrega | Baixa | Baixo| Testar previamente todos os recursos, ter backups (offline/online) e preparar um plano alternativo para apresentação/entrega.|
